Welcome to the Spring Security Authentication Examples!

This application demonstrates various authentication mechanisms supported by Spring Security. Each example is self-contained and can be explored independently.

Demo Users

The following demo users are pre-configured in the H2 database:

  • admin / password (Roles: ADMIN, USER)
  • user / password (Role: USER)
  • john / password (Role: USER)
  • jane / password (Roles: USER, MANAGER)
  • bob / password (Role: USER)

You can also access the H2 Console: H2 Console (JDBC URL: jdbc:h2:mem:authdb, Username: sa, Password: password)

Traditional form-based authentication with username and password stored in the database.

Features:

4. SAML 2.0 Login Disabled

Enterprise SSO using SAML 2.0 protocol with Identity Providers.

Use Cases:

  • Enterprise identity federation
  • Single Sign-On (SSO)
  • Integration with SAML IdPs

5. CAS Authentication Disabled

Central Authentication Server integration for enterprise SSO.

Features:

  • Centralized authentication
  • Single Sign-On
  • Ticket-based authentication

Note: Requires CAS server installation

6. JAAS Authentication Disabled

Java Authentication and Authorization Service integration.

Use Cases:

  • Legacy system integration
  • Custom LoginModules
  • JAAS policy configuration

Note: Legacy authentication mechanism

7. Pre-Authentication Disabled

Integration with external authentication systems like SiteMinder or Java EE security.

Features:

  • Request header authentication
  • Trust external authentication
  • Spring Security authorization

Note: Requires external authentication system

8. X.509 Authentication Disabled

Client certificate-based authentication using SSL/TLS mutual authentication.

Features:

  • Certificate-based authentication
  • SSL/TLS mutual auth
  • Strong cryptographic security

Note: Requires SSL configuration and client certificates
